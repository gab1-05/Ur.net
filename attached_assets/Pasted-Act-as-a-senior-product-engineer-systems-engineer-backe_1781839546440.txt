Act as a senior product engineer, systems engineer, backend engineer, frontend engineer, desktop app engineer, and UI/UX designer. Build a complete production-grade application called “Network Diagnostics Dashboard”.

==================================================
CHOSEN STACK — USE THIS EXACTLY
==================================================

Use this exact stack unless there is a hard technical blocker:

- Frontend: React + TypeScript + Vite
- Styling: Tailwind CSS with a custom design system
- Backend API: FastAPI
- Command execution + parsing: Python subprocess + parser modules
- Local persistence: SQLite
- Desktop packaging: Tauri
- Charts: Recharts
- Tables: TanStack Table
- Forms + validation: React Hook Form + Zod
- State/data fetching: React Query
- Routing: React Router
- Icons: Lucide
- Testing: pytest + Vitest + Playwright

Why this stack:
- Tauri uses the system WebView instead of bundling Chromium, which generally makes it lighter than Electron for a utility-style desktop app.
- FastAPI fits a typed API backend and is commonly run with ASGI servers such as Uvicorn.
- Python subprocess is appropriate for controlled system command execution and output parsing.

==================================================
CLI INTEGRATION RULES
==================================================

All command execution must be real and safe.

Use Python subprocess for controlled execution of supported commands such as:
- ping
- tracert / traceroute
- nslookup or dig
- ipconfig / ifconfig / ip
- arp
- route or equivalent
- netstat if useful for diagnostics

Non-negotiable command execution rules:
- Never execute arbitrary raw shell input
- Never pass unsanitized strings into shell execution
- Use a strict allowlist-based command registry
- Validate all target input
- Sanitize all parameters
- Add timeouts
- Add cancellation support where practical
- Capture stdout, stderr, and exit codes
- Parse outputs into structured JSON
- Separate command adapter logic from parser logic
- Detect Windows vs Linux vs macOS and choose the correct adapter automatically
- If a command is unsupported on a platform, show a proper unsupported state
